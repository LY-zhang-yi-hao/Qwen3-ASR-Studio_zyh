## 任务名称
输入法模式增加 Ctrl+空格按住录音、松开停止的快捷键

## 背景说明
- 输入法模式（画中画 PiP）目前通过界面按钮手动开始/停止录音。
- 希望在 PiP 窗口中支持 `Ctrl + 空格`：按下开始录音、松开即停止并触发识别，提升“语音输入法”体验。
- 需避免影响主界面已有的空格录音逻辑。

## 方案设计
- 仅在 PiP 窗口中注册快捷键监听，依赖 `window.documentPictureInPicture` 提供的子窗口上下文。
- 逻辑：keydown 侦测同时按下 `Ctrl` 与 `Space` 且未录音时开始录音；keyup 侦测 `Ctrl` 与 `Space` 松开且正在录音时停止并走现有识别流程。
- 通过 ref 标记按键状态，避免长按/重复触发；清理事件监听防止泄漏。
- 主窗口（非 PiP）保持现有空格录音逻辑不变。

## 实施步骤
1. 在 `PipView` 组件内添加快捷键监听逻辑与状态标记，绑定/解绑 `keydown`、`keyup`。
2. 将快捷键调用已有的 `startRecording` / `stopRecording` 流程，保持现有 UI 行为。
3. 更新任务日志记录方案与结果。

## 预期效果
- 在 PiP 窗口按住 `Ctrl+空格` 开始录音，松开任一键即停止并触发识别。
- 不影响主页面和其他输入场景；若浏览器不支持 PiP，行为不变。

## 最终结果
- `PipView` 组件新增 `Ctrl+空格` 快捷键监听，按下组合键自动开始录音，松开任意按键即停止并触发识别，使用 `useRef` 标记状态避免重复触发。
- 监听仅作用于 PiP 窗口，主界面原有空格录音逻辑保持不变。
